Conditional Malware Detection Through Emulation in ASP

By Aiden McClelland

This project is a prototype for a malware detector using ASP.
A full description of this project can be found in this report.
To try it out, download this zip file.
Once you have extracted the file, 

cd
into it and run 
./aspemu instance-6.ins
.
If this runs without errors, you are good to go! You can now write new problem instances and test them out.
If not, it is likely you do not meet all dependencies for the project.

Dependencies

  • Python >= 3.4.3
  • clingo >= 5.1.0


usage: ./aspemu  [timesteps]
If timesteps are not provided it will default to twice the maximum PC of the problem instance.


INS Format

  • Precede your program with the string 
    .code
    and nothing else on the line. Anything before this will be ignored.
  • Start each line with the PC of the current instruction in HEX with the 0x prefix, followed by a single space.
  • Instructions are to be written in all caps, and immediate values are to be written in unsigned HEX with the 0x prefix.
  • The only API Calls supported as of now are "GetSomething" (a placeholder) and "Exit".
